## Introduction
이번 글에서는 AWS 계정을 생성하고, EC2 인스턴스를 직접 만들어 SSH로 접속하는 과정을 실습합니다.
이전 글에서 다룬 과금 체계를 참고하여, **프리 티어(Free Tier)** 범위 안에서 진행합니다.

## 1. AWS 계정 생성

> 계정 생성 단계가 간단하며 여기서 다루지 않겠습니다. 

가입 시 필요한 정보는 다음과 같습니다.

- 이메일 주소 및 AWS 계정 이름
- 비밀번호
- 연락처 정보 (개인 또는 비즈니스)
- **신용카드 정보** — 프리 티어 사용 시에도 등록이 필요합니다. 실제 청구는 무료 한도 초과 시에만 발생합니다.
- 전화번호 인증 (OTP)

가입 완료 후 **AWS Management Console**에 루트 계정(Root User)으로 로그인합니다.
![루트 유저 로그인 화면](/images/cloud_prog_week1/aws_login.png)

> **보안 권장사항**: 루트 계정은 초기 설정 이후 일상적인 작업에 사용하지 않는 것이 좋습니다. IAM 사용자를 별도로 생성하여 사용하세요.

## 2. 프리 티어 주의사항

AWS 프리 티어는 신규 가입 후 **6개월간** 일부 서비스를 무료로 제공합니다. EC2 기준 무료 범위는 다음과 같습니다.
- 무료 사용 가능한 인스턴스: `t2.micro` 또는 `t3.micro`
- 사용 가능한 시간:         월 750시간
- EBS 스토리지:           30 GB(SSD)
- 데이터 전송:             월 10GB (아웃바운드)

**⚠️ 비용 발생 주의 상황**

- `t2.micro` 이외의 인스턴스 유형 선택 시
- 인스턴스를 **중지(Stop)** 하지 않고 방치할 경우 — 중지해도 EBS 요금은 계속 발생합니다!
- 탄력적 IP(Elastic IP)를 인스턴스에 연결하지 않은 채 보유할 경우
- 프리 티어 6개월 만료 후

> 실습 후에는 반드시 인스턴스를 **중지(Stop)** 하거나 **종료(Terminate)** 하세요.
> AWS Budgets에서 월 $1 알림을 설정해 두면 예상치 못한 과금을 방지할 수 있습니다.

## 3. EC2 인스턴스 생성

### 3-1. EC2 콘솔 접속

우측 상단 리전을 **`ap-northeast-2`** 로 변경합니다.
![set_region](/images/cloud_prog_week1/set_region.png)
AWS Management Console 상단 검색창에 **"EC2"** 를 검색하고 서비스로 진입합니다.

![search_ec2](/images/cloud_prog_week1/search_ec2.png)

**"인스턴스 시작(Launch Instance)"** 버튼을 클릭합니다.

---

### 3-2. 인스턴스 설정

**① 이름 및 태그**

인스턴스를 식별할 이름을 입력합니다. (예: `my-first-ec2`)

**② AMI (Amazon Machine Image) 선택**
![set_instance](/images/cloud_prog_week1/set_instance.png)

AMI가 무엇인지 궁금하시다면 간단하게 설명드리겠습니다.
우리가 인스턴스를 생성할 때 운영체제(Linux, Windows), 소프트웨어(Python, Node.js), 그리고 다양한 설정(패키지, 환경 설정, 권한 등)을 직접 구성할 수 있습니다. 
하지만 동일한 환경의 인스턴스를 여러 개 생성하려면 이러한 과정을 매번 반복해야 하기 때문에 번거롭습니다.
이럴 때 필요한 모든 설정을 하나의 AMI로 만들어 저장해두면, 해당 AMI를 기반으로 여러 개의 인스턴스를 쉽고 빠르게 생성할 수 있습니다

> 프리 티어 실습에는 아래를 권장합니다.

- **Amazon Linux 2023** — AWS 최적화, 프리 티어 적용
- **Ubuntu 24.04 LTS** — 범용적으로 많이 사용


**③ 인스턴스 유형 선택**
![instance_type](/images/cloud_prog_week1/instance_type.png)
![instance_type_exp](/images/cloud_prog_week1/instance_type_exp.png)

인스턴스를 컴퓨터라고 생각하면 인스턴스 유형은 컴퓨터의 사양이라고 생각하면 됩니다. 그래서 해당 소프트웨어를 실행할 때 얼마 만큼의 컴퓨팅 성능이 필요할지를 고민한 다음에 맞는 인스턴스 유형을 고를 수 있습니다. 

더 자세한 정보: [instance-types-explanation](https://docs.aws.amazon.com/ec2/latest/instancetypes/instance-types.html ) 

> free tier는 `t3.micro` 를 선택합니다.

**④ 키 페어(Key Pair) 생성**

SSH 접속에 사용할 키 페어를 생성합니다.

1. **"새 키 페어 생성"** 클릭
2. 이름 입력 (예: `my-ec2-key`)
3. 유형: **RSA**, 형식: **`.pem`** (Linux/Mac) 또는 **`.ppk`** (Windows PuTTY)
4. **"키 페어 생성"** 클릭 → `.pem` 파일이 자동으로 다운로드됩니다

![key_pair](/images/cloud_prog_week1/key_pair.png)

> `.pem` 파일은 **재다운로드가 불가능**합니다. 안전한 위치에 보관하세요.

**⑤ 네트워크 설정 (보안 그룹)**

기본 값으로 설정하면 됩니다.

> 소스를 `0.0.0.0/0`(전체 허용)으로 설정하면 보안에 취약합니다.

**⑥ 스토리지 구성**

기본값 **8GB gp3** 를 유지합니다. (프리 티어 30GB 이내)

---

### 3-3. 인스턴스 시작

모든 설정을 확인한 후 **"Launch Instance"** 버튼을 클릭합니다.


EC2 콘솔 → 인스턴스 목록에서 상태가 **`running`** 으로 바뀌면 준비 완료입니다.
인스턴스의 **퍼블릭 IPv4 주소**를 복사해 둡니다.

![인스턴스 시작 완료 화면](/images/cloud_prog_week1/running_instance.png)

## 4. SSH 접속

### Mac / Linux

터미널을 열고 다음 명령어를 실행합니다.
```bash
# 키 파일 권한 설정 (최초 1회)
chmod 400 ~/Downloads/my-ec2-key.pem

# SSH 접속
ssh -i ~/Downloads/my-ec2-key.pem ec2-user@<퍼블릭-IP>
```

> AMI에 따라 기본 사용자 이름이 다릅니다.
> - Amazon Linux: `ec2-user`
> - Ubuntu: `ubuntu`
> - Debian: `admin`

접속에 성공하면 아래와 같은 프롬프트가 표시됩니다.
```
   ,     #_
   ~\_  ####_        Amazon Linux 2023
  ~~  \_#####\
  ~~     \###|
  ~~       \#/ ___   https://aws.amazon.com/linux/amazon-linux-2023
   ~~       V~' '->
    ~~~         /
      ~~._.   _/
         _/ _/
       _/m/'
[ec2-user@ip-172-31-xx-xx ~]$
```

![SSH 접속 성공 화면](/images/cloud_prog_week1/ssh_connection.png)

### Windows (PowerShell)

Windows 10 이상에서는 PowerShell에서 동일한 `ssh` 명령어를 사용할 수 있습니다.
```powershell
ssh -i C:\Users\YourName\Downloads\my-ec2-key.pem ec2-user@<퍼블릭-IP>
```

## 5. 인스턴스 중지 및 정리

실습이 끝났다면 반드시 인스턴스를 정리합니다.

- **중지(Stop)**: 인스턴스를 일시 중단합니다. EBS 스토리지 요금은 계속 발생합니다.
- **종료(Terminate)**: 인스턴스와 EBS를 완전히 삭제합니다. 더 이상 사용하지 않는다면 이 옵션을 선택하세요.

EC2 콘솔 → 인스턴스 선택 → **"인스턴스 상태"** 드롭다운에서 원하는 작업을 선택합니다.

![인스턴스 중지/종료 화면](/images/cloud_prog_week1/instance_delete.png)
